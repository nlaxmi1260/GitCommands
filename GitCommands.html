Git Commands

Following command are most commonly used in devops daytoday life. In github.com login and create a repo .Create a new branch .

Commands Description
$ git clone -b [branchname] [urlof the repo]
e.g. git clone -b Feature-gitCommands git@github.com:nlaxmi1260/GitCommands.git		 This command is used to clone github repo with specific branchname and urloftheRepo to the local
$ git status This command is used to know the status of the repo, and also suggest the next cource of action.
$ git add .
$ git add [fileName] 		This command will add all the untraced files to traced files. git add filename will add specific file to the git
$ git commit -m "[Comments]" This command will commit the staged files.
$ git diff This command shows the difference from the last commit
$ git log This command shows the various commitIds
git config -- global --list Lists git global configurations
$ git reset --hard [commitId] This Command is used to save the changes to that commitId
$ git push This command is used to push local changes to github repo when github repo is configured with local git.
when github is not configured will give an error like no configured destination.
$ git remote -v This command will show the remote reference of the repository only when we configured with a repository. if we didnot configure with repo then remote reference will not be shown.
$ git remote add origin https://github.com/OWNER/REPOSITORY.git This command git remote add command takes two arguments:
A remote name, for example, origin
A remote URL, for example, https://github.com/OWNER/REPOSITORY.git
$ git remote -v To Verify new remote
origin https://github.com/OWNER/REPOSITORY.git (fetch)
origin https://github.com/OWNER/REPOSITORY.git (push)
$ git remote set-url origin git@github.com:OWNER/REPOSITORY.git Change your remote's URL from HTTPS to SSH with the git remote set-url command.
$ git branch This command shows the list of branches
$ git branch [branchname] This command will create a new branch with the branchName
$ git checkout [branchName] This command will change from one branch to another branch.
$ g git push --set-upstream origin Feature-GitCommands To push the current branch and set the remote as upstream
$ git fetch This command fetches the changes from remote to local
$ git merge This command merges changes from remote to local
$ git rebase
$ git stash This command is used to stash the commited changes
$ git stash list This command lists stashed commit IDs
$ git stash apply This command applies the stashed commitId to modified
$ git checkout [branchName] This command checkout to the given branchName
$ git branch -d [branchname] This command deletes the given branchname if already merged
$ git branch -D [branchname] This command forcefully deletes the given branchname which is not merged
$ git pull This command fetches and merges the remote changes to the local repo
$ git init This command is used to inialise empty git repo in local,which is used to track the changes ,for logging and every change
$ ls -a in .git folder hooks folder will be there, which is used for not sending the secure info to git.
